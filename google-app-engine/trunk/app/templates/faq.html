    What is this site?
    The brief description is that it's a site that grabs "topics that people are interested in" and creates photo galleries based on them. Currently it gets it's topics from the Twitter API and then searches for photos related to those topics using the Flickr API.
    The site seems pretty simple, why bother creating it?
    I'm a big fan of learning new things within the field of software development. As such, I enjoy picking languages, frameworks, and the like and building a project using them to get a feel for how they work. This project was the result of combined desire to learn about Python and one of the Platform as a Service platforms that exists.
    There's something I'd like to compain about, or see changed. How can I contact you?
    The best way to let me know about a problem/suggestion related to the site would be through the support page for it on Sourceforge
    I see the source for the site is available on Sourceforge....
    Indeed. This site, including it's current implementation and ones using other languages/frameworks are available via Sourceforge. Building it is a learning experience for me and making it available so others can use it to that same purpose seemed like a good idea.
    Why can't I put in my own search term and have a gallery displayed for it?
    Each time a gallery is ingested into the system, it requires a call to the Flickr API. Calls to that API are rate limited based on the app key used. I can't allow an unconstrained number of hits against the API without risk of impacting the normal operation of the site, especially since going over the rate disables the key permanantly.
    Why can't I search the existing galleries to see if there's one I'm interested in?
    Google App Engine doesn't have a real search engine (ie, SOLR/Lucene, etc) built in. It might be possible to build a search engine using what is available, but it will require more work than I can spare at the moment.
    Why can't we rate galleries so that the more interesting ones stay in the system longer?
    It's in the plan of things to do, actually. However, the ability to keep ratings on things requires database operations at rate that could surpass what Google App Engine allows. There is a way to make it work, but I haven't had time to look into it yet.
